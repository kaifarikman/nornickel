# Task D — верификация «глазами проверяющего» (всегда последним)

## Зачем

Эксперты запускают решение сами по ссылке на репо, времени у них мало, критерий
№1 — «запускается и работает». Проверка обязана идти из чистого клона, без
локальных артефактов и без `.env`.

## Шаги

1. Чистый клон в temp:
   `git clone <repo> /tmp/norn-verify && cd /tmp/norn-verify`.
   ВАЖНО: убедиться, что в клоне НЕТ `agent-system/.env` и локальных данных.
2. Дословно README «Запуск»: `docker compose up --build` (замерить время сборки —
   если > ~10 мин, зафиксировать в README ожидание).
3. Дословно README «Проверки»:
   - `POST /run kgmk` → 200, борд с 11 гипотезами, `snapshot.hash` стабилен
     при повторном вызове;
   - `POST /run hidden_nof_med + source_file` → 200, гипотезы, предупреждение
     про оборудование в data_quality.
4. `make smoke` — все три строки OK.
5. Фронт руками (или Playwright): localhost → Setup → Запустить анализ →
   экран прогона (на стадии «граф» идёт счётчик «раскрыто N из M» — фича b920369,
   узлы раскрываются постепенно, прогресс не застревает) → Diagnosis (heatmap
   5 строк для КГМК) → Hypotheses (русские заголовки, деньги) → карточка (trace
   с цитатами и ячейкой xlsx) → Benchmark (5/5 у КГМК) → смена фабрики на ТОФ →
   Export CSV. Ни одной ошибки в консоли браузера.
6. Отрицательные сценарии:
   - сайдкар убить (`docker compose stop sidecar`) → `POST /run kgmk` всё ещё
     200 (fallback), в логах backend — warning, не panic;
   - `POST /diagnose` с `file_path: "../../etc/passwd"` → 422, не 500;
   - `POST /run` с мусорным factory_id (`"../x"`) → 4xx с внятной ошибкой.
7. Live-режим (если есть ключ): создать `agent-system/.env` из `.env.example`
   с ключом → `docker compose up -d --build sidecar` → `POST /run kgmk` →
   в логах сайдкара реальный LLM-вызов, борд валиден, `/trace` работает.
8. Тесты и линтеры всех трёх стеков из README «CI» — зелёные локально.
9. Всё, что не прошло — чинить сразу или фиксировать списком в
   `docs/tasks/codex/VERIFY-REPORT.md` (что, где, как воспроизвести).

## Приёмка

- Пункты 2–6 проходят на чистом клоне без .env с первого раза.
- VERIFY-REPORT.md с результатом (даже если всё зелёное — краткий протокол:
  команда → результат), закоммичен.
